    Project Overview

    SpamCall (防騙衛士) is an advanced, native dual-platform security solution built for Hong Kong users. It intercepts spam phone calls, flags fraudulent text messages, and employs local heuristics to isolate conversational scams.

    Call Screen Blocking

    Automatically screening incoming calls. Uses system hooks to block blacklisted numbers silently while keeping detailed internal logging history.

    Skepticism Engine

    Evaluates transcripts and message content to identify contradictions of official procedures (e.g. asking to mail cards or requesting verification codes).

    Active Honeypot Defense

    Automatically sends SMS locator reply messages when phishing links arrive, trapping scammers by logging their IP addresses, devices, and geolocations.

    Structural Architecture

    System Architecture

    SpamCall connects deep OS-level intercepts with a local database logic and custom accessibility services to parse incoming calls and messages.

    Interception and Evaluation Pipeline

    Incoming calls or messages trigger native system services. The interceptors parse the identifiers, evaluate against the local database blacklist, and run speech/text checks via the Skepticism Engine. If a phishing link is found, a honeypot geolocation request is issued.

    // Intercept Pipeline Sequence
    1. System Hook [CallScreeningService / CallKit] catches incoming number.
    2. If number matches blacklisted item in Room/SQLite DB -> Block Call & Launch Overlay.
    3. If call accepted -> Accessibility Service/Speech Transcriber scrapes text dialogue.
    4. Skepticism Engine analyzes speech for contradiction patterns -> Triggers spoken Cantonese warnings if risk > 50%.
    5. If phishing link SMS received -> Automatically replies to sender with locator geocode honeypot.

    Honeypot IP Tracker

    When a link is caught, the app logs the event and sends back a locator link: https://guardcall.link/consent-{reportId}.

    Capturing Geolocation details.

    Consent Tracking Capture

    When clicked, the honeypot page queries browser geodetic APIs. The server logs the scammer's details:

    • IP Address (e.g. 42.200.180.12)
    • Geo Location (Sham Shui Po, HK)
    • Network Carrier (China Mobile HK)
    • User-Agent (OS details / device)
    Algorithmic Brain

    Skepticism Engine

    The analysis engine runs localized scans against payment terms, common spam patterns, and official organizational rules.

    Risk Calculation Layers

    The total risk index is accumulated across three layers and capped at 100%. If a violation of an official procedure rule occurs, a severe penalty of 60 points is added.

    Risk Index = Min(100, Score_Payments + Score_Patterns + Score_Contradictions)

    Protected Payments (+15 pts)

    Scanning for mentions of critical money endpoints:

    • Bank Apps & Internet Banking ("網上銀行", "手機銀行")
    • FPS / Faster Payment ("轉數快")
    • Octopus, Alipay, WeChat Pay ("八達通", "支付寶")

    Procedure Violations (+60 pts)

    Checks contradictions against what institutions never do:

    • HK Police: Will NEVER request you to mail a physical bank card.
    • Alipay: Will NEVER request verification codes to cancel fees.
    • HSBC: Will NEVER send links asking for security codes.

    Native OS Hooks

    Device Integrations

    Explore the technical stack layout for Android (Kotlin) and iOS (Swift) integrations.

    Android Details
    • CallScreeningServiceImpl hooks incoming dial events asynchronously, comparing numbers with Room databases.
    • LiveCallAccessibilityService scans screen node hierarchies and initializes speech recognizers using a Cantonese locale (zh-HK).
    • SmsReceiver extracts links, logs events, and fires automated honeypot responses.
    iOS Details
    • CallDirectoryHandler utilizes CallKit to pre-compile and sort blacklists in ascending order.
    • MessageFilterExtension uses IdentityLookup to analyze text bodies and send scam SMS to system "Junk" folders.
    • SwiftData / SQLite handles offline DB queries on device targets.
